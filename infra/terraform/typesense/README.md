# Typesense Cluster Infrastructure (Terraform)

This module provisions a production-grade Typesense deployment for `dev`, `staging`, and `prod` on shared EKS.

## What it creates

- 3-node Typesense StatefulSet (`1 leader + 2 replicas`)
- Node resources: `4 vCPU / 8Gi RAM` per node
- Persistent volumes (EBS via `gp3` StorageClass)
- Cluster-internal service (`ClusterIP`) and private traffic policy
- IRSA role for Typesense service account (Secrets Manager + S3 + CloudWatch)
- Secrets Manager secret for Typesense API key + optional 30-day rotation
- S3 backup bucket (versioned + encrypted)
- CloudWatch dashboard + alarms for latency, replication lag, and errors

## Deploy

```bash
cd infra/terraform/typesense
terraform init
terraform apply -var-file=environments/dev.tfvars
terraform apply -var-file=environments/staging.tfvars
terraform apply -var-file=environments/prod.tfvars
```

> Replace `EXAMPLE` OIDC values in each tfvars file before apply.

## Backup/Restore cadence

- Hourly snapshots are written to the environment S3 backup bucket.
- Weekly restore tests are executed per the runbook: `docs/runbooks/typesense-cluster.md`.

## Notes

- API key is generated by Terraform and stored in AWS Secrets Manager.
- To enable automated key rotation without downtime, provide `typesense_api_key_rotation_lambda_arn`.
